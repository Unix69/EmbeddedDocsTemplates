137 <p>In your source code, you can use these custom aliases in comments. Doxygen will render them with icons and labels.</p>
138
139 <ul style="list-style-type: none; padding-left: 0; font-size: 18px; line-height: 2;">
140 <li>üõ†Ô∏è <b>@fix</b> ‚Äì Use for code fixes or patches.</li>
141 <li>üêû <b>@bug</b> ‚Äì Use to document a bug found or fixed.</li>
142 <li>üîÑ <b>@change</b> ‚Äì Use to describe a change in functionality or logic.</li>
143 <li>üìù <b>@todo</b> ‚Äì Use to mark something that needs to be done.</li>
144 <li>üîó <b>@api</b> ‚Äì Use to highlight API-related notes.</li>
145 <li>üì¶ <b>@namespace</b> ‚Äì Use to describe a namespace.</li>
146 <li>‚ú® <b>@feature</b> ‚Äì Use to document a new feature.</li>
147 </ul>

217 <h2><img src="Images/Logo/doxygen.svg" alt="Doxygen" width="24px" height="24px"> Doxygen Customization</h2>
218 <b>README Template</b> use doxygen customization as follow:
219 <br>
220 <ul style="list-style-type: none; padding-left: 0;">
221 <li>‚úÖ HTML output enabled (<code>GENERATE_HTML=YES</code>)</li>
222 <li>‚úÖ Tree view in HTML enabled (<code>GENERATE_TREEVIEW=YES</code>)</li>
223 <li>‚úÖ Search engine enabled (<code>SEARCHENGINE=YES</code>)</li>
224 <li>‚úÖ Markdown supported as main page and in input (<code>MARKDOWN_SUPPORT=YES</code>)</li>
225 <li>‚úÖ Graphs enabled (Class, Collaboration, Include, Caller/Call Graphs, UML, Directory, Group)
226 (<code>CLASS_GRAPH, COLLABORATION_GRAPH, INCLUDE_GRAPH, CALL_GRAPH, CALLER_GRAPH, UML_LOOK, GROUP_GRAPHS, DIRECTORY_GRAPH=YES</code>)</li>
227 <li>‚úÖ Graphviz interactive SVG with multi-targets, transparent background, and cleanup (<code>INTERACTIVE_SVG, DOT_MULTI_TARGETS, DOT_TRANSPARENT, DOT_CLEANUP=YES</code>)</li>
228 <li>‚úÖ Warnings logged to <code>docs/doxygen_warnings.log</code> (<code>WARN_LOGFILE</code>)</li>
229 <li>‚úÖ Source browser and inline sources enabled (<code>SOURCE_BROWSER=YES, INLINE_SOURCES=YES</code>)</li>
230 <li>‚úÖ Extract all entities including private, static, local classes/members, packages, local methods, anonymous namespaces (<code>EXTRACT_ALL, EXTRACT_PRIVATE, EXTRACT_STATIC, EXTRACT_LOCAL_CLASSES, EXTRACT_PACKAGE, EXTRACT_LOCAL_METHODS, EXTRACT_ANON_NSPACES=YES</code>)</li>
231 <li>‚úÖ Undocumented classes/members are shown (<code>HIDE_UNDOC_CLASSES=NO, HIDE_UNDOC_MEMBERS=NO</code>)</li>
232 <li>‚úÖ Sorting & organization: sorted by member, brief, and scope, section labels, index generated (<code>SORT_MEMBER_DOCS, SORT_BRIEF_DOCS, SORT_BY_SCOPE_NAME, ENABLE_SECTION_LABELS, GENERATE_INDEX=YES</code>)</li>
233 <li>‚úÖ HTML customization: header, footer, extra stylesheets (<code>HTML_HEADER, HTML_FOOTER, HTML_STYLESHEET, HTML_EXTRA_STYLESHEET</code>)</li>
234 <li>‚úÖ MathJax support for formulas (<code>USE_MATHJAX=YES, FORMULA_FONTSIZE=12</code>)</li>
235 <li>‚úÖ Autolinking support (<code>AUTOLINK_SUPPORT=YES</code>)</li>
236 <li>‚úÖ TOC includes headings up to level 3 (<code>TOC_INCLUDE_HEADINGS=3</code>)</li>
237 <li>‚úÖ Manual pages generation enabled (<code>GENERATE_MAN=YES</code>)</li>
238 <li>‚úÖ Enumerations displayed one value per line (<code>ENUM_VALUES_PER_LINE=1</code>)</li>
239 <li>‚úÖ Example files included recursively (<code>EXAMPLE_PATH, EXAMPLE_RECURSIVE=YES</code>)</li>
240 <li>‚úÖ References and relations shown (<code>REFERENCED_BY_RELATION=YES, REFERENCES_RELATION=YES</code>)</li>
241 <li>‚úÖ Verbatim headers displayed (<code>VERBATIM_HEADERS=YES</code>)</li>
242</ul>

247 <h2> ‚úÖ Advantages</h2>
248 The <b>README Template</b> offers multiple advantages for software documentation and maintenance:
249 <ul>
250 <li>üîó <strong>Complete Traceability:</strong> Every change in the code or functionality is documented and linked to the corresponding version.</li>
251 <li>‚öôÔ∏è <strong>Automation:</strong> Make + Doxygen toolchain automatically updates HTML documentation with Markdown content and inline code comments.</li>
252 <li>üß© <strong>Integration:</strong> Doxygen tags (@fix, @bug, @change, @todo, @api, @namespace, @feature) link development notes directly to source code.</li>
253 <li>üìÇ <strong>Modular Clarity:</strong> Folder structure (Version, Usage) separates concepts and production phases, making documentation readable and navigable.</li>
254 <li>ü§ù <strong>Collaboration-Friendly:</strong> Multiple team members can edit Markdown files, while Doxygen generates a unified, always up-to-date HTML site.</li>
255 <li>‚è±Ô∏è <strong>Up-to-Date Documentation:</strong> Changes in code or project structure automatically propagate to the documentation without manual intervention.</li>
256 <li>üìè <strong>Standardization:</strong> Provides a consistent template to structure and maintain project documentation across multiple projects.</li>
257 <li>üåê <strong>Interactive HTML Site:</strong> HTML-based site mirrors Markdown organization with navigation tree, search, and clickable references.</li>
258 </ul>
