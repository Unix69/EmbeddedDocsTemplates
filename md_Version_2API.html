Characteristics

Each API defines:

  • a set of Parameters to specify during the API Request.
  • a Response Type returned on API Response. Both depend on the given API implementation specifications so read the single API documentation to know how each API act and how to use it.


ğŸ® Used by

APIs are used by:

  • Features to implement themself. Each API declare into which features they are used.
  • Actors into Use Cases to perform tasks. Each API declare for which actors they are suited for.

â­ Versions

Each API is available on a specific Version and can contain Bugs solved as soon as possible by Fixes. Each API release follows the Release Policy and declares the implemented versions, the discovered bugs, and provided fixes.

ğŸ§© Collection

APIs are collected into Namespaces that expose the API and other related resources to use the API properly. Each API declares the Namespace that contains it.

ğŸ” Accessibility

API's accessibility is Role based. If the role used by the actor to access the API does not correspond to the admitted, the API will fail and return a permission fail response. Each API declares the admitted roles.
