Namespace

The whole set of [PROJECT_NAME] APIs is divided into Namespaces. Each Namespace is composed by some Sub-Namespaces that specializing it. Every Namespace provides APIs that implements Features. A Namespace is characterized by some attributes and properties. A Namespace can be modified by some Namespace Changes, during [PROJECT_NAME] lifecycle.

An Attribute is a Read Only (RO) field, representing the Namespace characteristic that cannot be changed by the namespace user. While a Property is a RO or Readable and Writeable (RW) field, representing the Namespace characteristic that can be changed by the namespace user.

Furthermore every Namespace can contain Namespace Bugs that will be resolved by Namespace Fixes.

[PROJECT_NAME] Namespaces are collected into the Namespace Index section of this document and each of them is characterized by some attributes, that globally define the Namespace itself. This attributes are listed into the table below
