\begin{DoxyItemize}
\item {\itshape Features} to implement themself. Each A\+PI declare into which features they are used.
\item {\itshape Actors} into {\itshape Use Cases} to perform tasks. Each A\+PI declare for which actors they are suited for.
\end{DoxyItemize}\hypertarget{md_Version_API_autotoc_md158}{}\doxysubsection{‚≠ê Versions}\label{md_Version_API_autotoc_md158}
Each A\+PI is available on a specific {\itshape Version} and can contain {\itshape Bugs} solved as soon as possible by {\itshape Fixes}. Each A\+PI release follows the  \href{RELEASE_POLICY.md}{\texttt{ {\bfseries{Release Policy}}}}  and declares the implemented versions, the discovered bugs, and provided fixes.\hypertarget{md_Version_API_autotoc_md159}{}\doxysubsection{üß© Collection}\label{md_Version_API_autotoc_md159}
A\+P\+Is are collected into {\itshape Namespaces} that expose the A\+PI and other related resources to use the A\+PI properly. Each A\+PI declares the {\itshape Namespace} that contains it.\hypertarget{md_Version_API_autotoc_md160}{}\doxysubsection{üîê Accessibility}\label{md_Version_API_autotoc_md160}
A\+PI\textquotesingle{}s accessibility is {\itshape Role} based. If the role used by the actor to access the A\+PI does not correspond to the admitted, the A\+PI will fail and return a permission fail response. Each A\+PI declares the admitted roles.\hypertarget{md_Version_API_autotoc_md161}{}\doxysubsection{‚Ü©Ô∏è Changes}\label{md_Version_API_autotoc_md161}
