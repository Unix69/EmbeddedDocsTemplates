Each A\+PI defines\+:
\begin{DoxyItemize}
\item a set of {\itshape Parameters} to specify during the {\bfseries{A\+PI Request}}.
\item a {\itshape Response Type} returned on {\bfseries{A\+PI Response}}. Both depend on the given A\+PI {\bfseries{implementation specifications}} so read the single {\bfseries{A\+PI documentation}} to know how each A\+PI act and how to use it.
\end{DoxyItemize}

~\newline
\hypertarget{md_Version_API_autotoc_md113}{}\doxysubsection{üéÆ Used by}\label{md_Version_API_autotoc_md113}
A\+P\+Is are used by\+:
\begin{DoxyItemize}
\item {\itshape Features} to implement themself. Each A\+PI declare into which features they are used.
\item {\itshape Actors} into {\itshape Use Cases} to perform tasks. Each A\+PI declare for which actors they are suited for. 
\end{DoxyItemize}\hypertarget{md_Version_API_autotoc_md114}{}\doxysubsection{‚≠ê Versions}\label{md_Version_API_autotoc_md114}
Each A\+PI is available on a specific {\itshape Version} and can contain {\itshape Bugs} solved by as soon as possible by {\itshape Fixes}. Each A\+PI release follow the {\bfseries{Release Policy}} and declare the implemented versions, the discovered bugs and provided fixes. \hypertarget{md_Version_API_autotoc_md115}{}\doxysubsection{üß© Collection}\label{md_Version_API_autotoc_md115}
A\+P\+Is are colleted into {\itshape Namespaces} that expose the A\+PI and other related resources to use the A\+PI properly. Each A\+PI declare the {\itshape Namespace} that contain themself. \hypertarget{md_Version_API_autotoc_md116}{}\doxysubsection{üîê Accessibility}\label{md_Version_API_autotoc_md116}
A\+PI\textquotesingle{}s accessibility is {\itshape Role} based, if the role used by the actor to access the A\+PI does not correspond to the admitted, the A\+PI will fail and return a permission fail respose. Each A\+PI declare the admitted roles. \hypertarget{md_Version_API_autotoc_md117}{}\doxysubsection{‚Ü©Ô∏è Changes}\label{md_Version_API_autotoc_md117}
An {\itshape A\+PI} can be modified by some {\itshape Changes}, during the \mbox{[}P\+R\+O\+J\+E\+C\+T\+\_\+\+N\+A\+ME\mbox{]} lifecycle so read the {\bfseries{Change Log}} to be updated with last A\+PI changes. \hypertarget{md_Version_API_autotoc_md118}{}\doxysubsection{üõ†Ô∏è Extendabilty}\label{md_Version_API_autotoc_md118}
It\textquotesingle{}s possibile to extend the set of A\+P\+Is of each namespace by adding new A\+PI. To know how, follow the {\bfseries{Development Guide}}.
