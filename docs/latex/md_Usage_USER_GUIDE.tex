This section explains the {\bfseries{main interactions for end-\/users}} of the R\+E\+A\+D\+ME Template. Users can explore the template structure, navigate Markdown documentation, and understand how documentation integrates with Doxygen-\/generated H\+T\+ML.

Key points\+:


\begin{DoxyItemize}
\item The template provides a {\bfseries{standardized layout}} for documenting projects.
\item Markdown files are {\bfseries{modular}}, representing features, versions, A\+PI, namespaces, bugs, fixes, and guides.
\item Users can quickly navigate {\bfseries{User Guide}}, {\bfseries{Admin Guide}}, and {\bfseries{Developer Guide}} using the Doxygen H\+T\+ML interface.
\end{DoxyItemize}

Operations that a user can perform with the R\+E\+A\+D\+ME Template\+:


\begin{DoxyEnumerate}
\item Explore the {\bfseries{directory structure}} to understand where each type of documentation is located ({\ttfamily Version/}, {\ttfamily Usage/} folders).
\item Open Markdown files in a text editor to read or modify content.
\item Use the {\bfseries{Makefile}} to generate the Doxygen H\+T\+ML documentation ({\ttfamily make build} ‚Üí {\ttfamily make doc}).
\item Navigate the {\bfseries{interactive H\+T\+ML site}} to explore links between Markdown files and source code.
\item Track {\bfseries{features, A\+PI endpoints, versions, bugs, and fixes}} directly in the H\+T\+ML interface.
\item Access {\bfseries{contact and project information}} for collaboration.
\end{DoxyEnumerate}
