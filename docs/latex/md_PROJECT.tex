\hypertarget{md_PROJECT_autotoc_md15}{}\doxysubsection{H\+T\+M\+L-\/based site}\label{md_PROJECT_autotoc_md15}
 The H\+T\+M\+L-\/based site consists of {\ttfamily .html}, {\ttfamily .js}, and {\ttfamily .css} files, {\bfseries{rooted at the base directory {\ttfamily ./}}}, that customize and extend the default Doxygen output. This layer transforms the Markdown documentation and source code comments into a fully navigable and interactive website.

The Doxygen part generates a {\bfseries{fully navigable H\+T\+ML site}} with integrated Markdown documentation\+:


\begin{DoxyItemize}
\item Markdown files are converted into H\+T\+ML pages within the site.
\item Source code documentation is included, showing classes, packages, file hierarchies, and diagrams.
\item Custom headers, footers, C\+SS, and JS allow a tailored visual style and interactive navigation.
\item The template supports {\bfseries{custom Doxygen tags}} ({\ttfamily @fix}, {\ttfamily @bug}, {\ttfamily @change}, {\ttfamily @todo}, {\ttfamily @api}, {\ttfamily @namespace}, {\ttfamily @feature}) for embedding comments directly in the code, keeping documentation always linked to the implementation.
\end{DoxyItemize}

\doxyparagraph*{Functionality and Structure}




\begin{DoxyItemize}

\hypertarget{md_PROJECT_autotoc_md16}{}\doxysubsubsection{How Doxygen Works in This Template}\label{md_PROJECT_autotoc_md16}

\begin{DoxyEnumerate}
\item {\bfseries{Configuration File ({\ttfamily \mbox{\hyperlink{doxygen_8ini}{doxygen.\+ini}}})}} ~\newline
 Contains project-\/specific parameters that are read by the script {\ttfamily \mbox{\hyperlink{doxygen_8sh}{doxygen.\+sh}}}. These parameters are used to configure Doxygen dynamically.
\item {\bfseries{Script ({\ttfamily \mbox{\hyperlink{doxygen_8sh}{doxygen.\+sh}}})}} Reads {\ttfamily \mbox{\hyperlink{doxygen_8ini}{doxygen.\+ini}}} and generates a {\ttfamily Doxyfile} by adding or modifying the necessary parameters. ~\newline
 This ensures that the H\+T\+ML documentation will reflect the current project setup, Markdown files, and source code structure.
\item {\bfseries{Generating Documentation}} ~\newline

\begin{DoxyItemize}
\item {\ttfamily make build}\+: prepares any required intermediate files or configuration.
\item {\ttfamily make doc}\+: runs Doxygen with the generated {\ttfamily Doxyfile} ({\ttfamily doxygen Doxyfile}) and produces the full H\+T\+ML documentation.
\item During this process, the Make command also {\bfseries{copies resources}} such as images into {\ttfamily docs/html} so that all links in Markdown and H\+T\+ML pages remain valid and functional.
\end{DoxyItemize}
\end{DoxyEnumerate}
